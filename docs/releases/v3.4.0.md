# OrbitTest v3.4.0

OrbitTest v3.4.0 adds the first publish-ready mobile testing layer for Android automation while keeping the existing browser runner flow intact.

## Highlights

- Added mobile test configuration through `use.mobile`, with optional `@orbittest/mobile` provider loading and `.env` support for ADB/device settings.
- Added Android automation APIs for app install/uninstall, launch/stop, touch gestures, keyboard input, UIAutomator selectors, screenshots, logcat, device state, and raw ADB access.
- Added mobile-aware test contexts with `page`/`web` for browser automation and `orbit`/`mobile` for Android automation.
- Added mobile assertions: `expect(orbit).toHaveText()`, `toHaveId()`, and `toMatchScreenshot()`.
- Added `orbittest devices` and `orbittest doctor` for connected Android device discovery and environment diagnostics.
- Added mobile failure/report artifacts, including screenshots, UI XML/JSON, logcat output, device/app metadata, and HTML report evidence.
- Added the `orbittest/mobile` export and TypeScript declarations for mobile-first tests.
- Added the publishable `packages/mobile` provider package with ADB, UIAutomator parsing, selector matching, artifacts, visual comparison, and doctor utilities.

## Packaging

- The root `orbittest` package now includes the mobile provider files needed by `orbittest/mobile`.
- Mobile provider runtime dependencies are declared at the root package level so the bundled mobile export resolves correctly after npm install.
- Added packaging tests to prevent the mobile export from regressing in future releases.

## Validation

- Unit coverage includes mobile parser/provider foundations and the root mobile export packaging guard.
- `npm pack --dry-run` verifies that `mobile.js`, `mobile.d.ts`, `core/providers/mobile.js`, and `packages/mobile/**` are included in the root package.
